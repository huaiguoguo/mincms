MinCMS是一个很酷的文章管理系统
在未正式发布的情况下 请勿使用本系统于生产环境

为什么要开发一个新的CMS？

目前的CMS 不管你需不需要的功能它都会给你，

而drupal相对性能不是很好


它是一个纯文章管理系统吗？

显然它可以用于交互型社区网站，在我们正式发布前，我们将基于该CMS开发一个社区网站。



何时发布？

目前我们没有确定的时间。

为了确保该系统正式发布时，出现尽量少的BUG，我们一直在努力用于生产环境下的项目，并不断优化。

直到我们确定该系统非常稳定，才会发布正式版。

GIT上的代码一般什么时间更新？
重要功能修复 或新功能添加后会在GIT上每月更新
